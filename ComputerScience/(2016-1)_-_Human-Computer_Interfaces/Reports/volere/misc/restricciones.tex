\section{Restricciones de la solución}
%formas de que el problema puede ser resuelto.
\begin{itemize}
  \item El proyecto debe ser enviado con el código fuente, la documentación, el ejecutable y la descripción del proyecto.
  \item Un proyecto no puede ser modificado después de ser aceptado.
  \item Un comentario no puede ser modificado posterior a su moderación.
  \item Una cuenta de usuario no puede ser eliminada pero si deshabilitada.
  \item Un proyecto publicado no puede ser eliminado.
  \item No existe un lapsus de tiempo fijo para subir proyectos.
  \item El idioma del producto es español.
  \item El sistema debe contar con un control de acceso.
  \item El proyecto no puede contar con un control de versiones.
\end{itemize}

\section{Ambiente de implementación de la solución}
%entorno físico y tecnológico en el que el producto será instalado.
\begin{itemize}
  \item El producto debe ser multiplataforma %(Roa dijo: El producto es una aplicación web)
  \item El formato de compresión de los archivos a subir a un proyecto debe ser estándar (rar, tar, zip).
\end{itemize}
